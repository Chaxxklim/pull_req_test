Author: John Kirkham <jakirkham@gmail.com>
Date:   Fri Dec 11 12:14:44 2020 -0800

    Request bot to check solvable before opening PRs
    
    As `distributed` package builds depend on `dask-core` package builds, if
    PRs are opened to both feedstocks to update the package version at the
    same time, the PR to `distributed` will fail until `dask-core` packages
    are available. This fixes that issue by ensuring that `distributed`'s
    dependencies (`dask-core` in this case) are available before opening a
    PR. That way the PR will only be opened after `dask-core` has new
    packages published. Should cutdown the time recipe maintainers spend
    restarting CI to get the new dependencies.
